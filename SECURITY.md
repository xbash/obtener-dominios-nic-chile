# Security Policy

## Supported Versions

Se mantiene soporte para la version actual de la rama principal y sus archivos
documentados en este repositorio.

## How to Report a Vulnerability

No reportes vulnerabilidades por issues publicos.

Para reportes privados, usa el canal privado que el mantenedor defina antes de
publicar el repositorio. No incluyas detalles explotables en issues publicos.

Incluye, si es posible:

- descripcion del problema
- archivo o modulo afectado
- pasos para reproducir
- impacto esperado
- evidencia minima o PoC

## Response Expectations

- Confirmacion de recepcion: dentro de 3 dias habiles.
- Triage inicial: lo antes posible despues de recibir el reporte.
- Si el hallazgo requiere correccion, se coordinara un parche antes de divulgar
  detalles publicamente.

## Scope

Este repositorio incluye scripts de consulta a NIC Chile, archivos de datos y
documentacion asociada.

## Safe Disclosure

Hasta resolver el problema, evita publicar capturas, URLs completas o detalles
operativos en canales publicos.
